Команды, которые я использовал в этом видео:

git config команда позволяет просматривать и настраивать параметры, контролирующие все аспекты работы Git
// git config --global user.name или user.email

git status отображает состояние рабочего каталога и раздела проиндексированных файлов

git init - команда создаёт в текущем каталоге новый подкаталог с именем .git

git add добавляет содержимое рабочего каталога в индекс (staging area) для последующего коммита (-a - добавить все)

git commit используется для коммита изменений в файлах проекта

git remote add используется для создания записи о новом подключении к удаленному репозиторию

git branch команда для создание / переключения веток

git push команда для публикации выгружаемых локальных изменений в центральном репозитории

git clone команда создаёт каталог, инициализирует в нём подкаталог .git, скачивает все данные из онлайн репозитория

git pull используется для извлечения и загрузки содержимого из удаленного репозитория и немедленного обновления локального репозитория этим содержимым

.gitignore игнорирование файлов *[oa] - заканчивающиеся на o и a
Символ (*) соответствует 0 или более символам; последовательность [abc] — любому символу из указанных в скобках (в данном примере a, b или c); знак вопроса (?) соответствует одному символу; и квадратные скобки, в которые заключены символы, разделённые дефисом ([0-9]), соответствуют любому символу из интервала (в данном случае от 0 до 9)

Пустые строки, а также строки, начинающиеся с #, игнорируются. - можно использовать, как комментарий

git diff - полная информация того, что поменялось

git push origin main - загрузка в репозиторий

git rm - удаление файла из индекса
--cached - удаление файла из индексации, но сохранение на ЖД
git rm --cached --ignore-unmatch *название файла* - удаление файла из репозитория облака
git mv - переименование

git remote - Просмотр удалённых репозиториев
-v - просмотр всех удаленных репозиториев в том числе клонированных

git remote add <shortname> <url> - присвоить репозиторию короткое имя
git fetch <shortname> - получить файлы у присвоенного репозитория
git pull - автоматическое получение изменений и слияние со своей версией
git remote rename - переименование репозитория
git tag (-l) - просмотр тегов о версии
-a - создать тег
-d - удалить тег
